Name

ipsvd-instruct - format of the ipsvd(8) instructions directory

Description

The internet protocol service daemons, ipsvd(8), can be told to read and follow instructions from a directory on incoming connections to the socket they listen on.

For mostly static instructions or for performance reasons, it is possible to compile the instructions from a directory into a constant database (cdb) with ipsvd-cdb(8) for faster lookup, and to tell ipsvd(8) to read the instructions from there.

Matching

On each incoming connection, the ipsvd(8) matches the client's IP address against files in the instructions directory. For example, the IP address a.b.c.d is matched against the following files in the instructions directory, in this order, first match wins:
    .
  1. a.b.c.d
    2. .
  2. a.b.c
    3. .
  3. a.b
    4. .
  4. a

After successfully matching a client's IP address against the instructions directory, ipsvd(8) examines the file that matched the IP address, and acts accordingly:

    .
  1. If the file doesn't have any of the user's access permissions set (chmod 0), the connection is closed immediately.
    2. .
  2. If the file has the user's execute permission set, ipsvd(8) reads the contents of the file and runs /bin/sh -c '<contents>' instead of the default program prog given at the command line for this connection.
    3. .
  3. If the file has the user's read permission set, ipsvd(8) reads the contents of the file and interprets each line as an instruction for this connection (see below).

If the client's IP address doesn't match any file in the instructions directory, the default action is taken (the program prog is run to handle the connection, unless the -n option is given).

Instructions

If ipsvd(8) is given instructions for an incoming connection, it reads the corresponding file and interprets each line as follows. The file may be empty, meaning that there is no special instruction.
    .
  1. If the line starts with a plus (``+''), and the string following the plus contains a ``='', ipsvd(8) puts the string following the plus into the environment before starting prog to handle the connection. If the string following the plus doesn't contain a ``='', ipsvd(8) prints a warning and discards this line.

If ipsvd(8) cannot interpret a line, it prints a warning, discards the line, and continues with the next instruction if any.

Example Instructions

+MEMORY=20000
This instruction causes the environment variable ``MEMORY'' with the value ``20000'' to be availbale to the program prog that handles the connection.
+DEBUG=
This instruction adds the variable ``DEBUG'' with an empty value to the environment.
